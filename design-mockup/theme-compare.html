유저 선택 시점

온보딩 3화면 끝, 초대 직전. 두 모드를 미리보기 카드로 나란히 보여주고 "어떤 동기가 네게 맞나" 단일 선택. 언제든 프로필에서 변경 가능. DB 필드: profiles.theme ('heat' | 'clarity', default 'clarity').

왜 같은 구조를 유지하나

운영·유지보수 비용 최소화. 기능·정책·스코어링·DB 스키마 100% 공유. 차이는 CSS 변수 덩어리 하나. 유저가 모드를 바꾸면 즉시 전환. 데이터·진행 상황 전부 그대로 유지.

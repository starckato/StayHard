Prototype · 캐릭터룸 ↔ 큐브 통합 — 4 옵션 비교

사용자 vision: "캐릭터가 큐브를 섭취 + 황금큐브 누적 → 티어 진화"
실제 코드 미반영. 시각 가안만. 어느 방향이 맞는지 골라줘 — 결정 후 구현 들어감. 가정: tier2 (각성자) · gold 32개 · silver 24개 · 다음 티어까지 54%.
OPTION A

바닥 더미 (Floor Pile)

큐브가 룸 바닥 한쪽 코너에 더미로 쌓임. 늘어날수록 더미 높이 증가. 캐릭터는 그 옆에 서있음 — "내가 모은 자국".
★ 각성자 ✕
🟡32 · ⚪24
  • + 시각적으로 가장 직관 — "내가 쌓은 것"
  • + 큐브 섭취 애니 도착점이 더미 (현재 캐릭터 도착)
  • − 100+ 큐브 시 stack 정리 알고리즘 필요
  • − 룸 일러스트 절반 가림
OPTION B

진열장 (Display Case / Trophy Wall)

룸 벽에 격자 선반. 각 칸 = 1 큐브. 100 칸 격자 가득 차는 게 1년 목표 — 박물관·트로피룸 느낌.
★ 각성자 ✕
SHELF · 56/100
  • + 빈 칸이 다음 목표 시각화 — "다음 칸 채워라"
  • + 1년치까지 일정 격자로 깔끔
  • − 비어있는 칸이 D0-D7 신규에겐 압박감
  • − 1000+ 큐브 시 격자 부족
OPTION C

흩어짐 + 인터랙션 (Scattered Floor)

큐브가 룸 바닥에 자연스럽게 흩어짐. 캐릭터가 그 사이를 walking. 정렬 안 된 "사용자가 만든 흔적들" 톤.
★ 각성자 ✕
  • + 가장 organic. "기록은 살아있다" 느낌
  • + red 큐브도 자연스럽게 섞임 (직시)
  • − 갯수 정확한 카운트 부재 (시각 추정만)
  • − 50+ 큐브 시 visual chaos
OPTION D

인벤토리 패널 (Stat Panel)

룸은 그대로 (캐릭터 + 룸 일러스트). 하단 패널에 큐브 카운트 + 다음 티어 progress 명시. 시각보다 데이터 중심.

VISION 정합성 체크

"캐릭터가 큐브를 섭취" — 큐브 popover 가 룸 진입 시에도 캐릭터로 빨려들어가는 애니는 모든 옵션에서 동일하게 동작. 단, 도착점이 다름:
• Option A: 큐브가 캐릭터 → 더미로 굴러 떨어짐
• Option B: 큐브가 캐릭터 → 격자 빈 칸으로 fly-in
• Option C: 큐브가 캐릭터 → 바닥에 떨궈짐 (랜덤 위치)
• Option D: 큐브가 캐릭터 → 카운터 숫자 bump 만

골랐어? A · B · C · D 중 한 글자만 알려주거나, 혼합/수정 의견 줘.
현재 상태: 코드 변경 0건, prototype 만 별도 페이지.
